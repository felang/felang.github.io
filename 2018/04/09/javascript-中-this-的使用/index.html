javascript 中 this 的使用

Posted on 2018-04-09 | In Javascript

绑定规则

默认绑定

即独立函数调用时 this 的绑定对象 

function foo() {
    console.log( this.a )
}
var a = a;
foo();  // 2

默认绑定中 this 指向全局对象, 所以这里调用 foo() 函数会将 this.a 解析为全局变量 a.

隐式绑定

当函数调用具有上下文对象时, 或函数属于某个对象或被包含时, this 指向这个函数的上下午对象.

function foo() {
    console.log( this.a )
}
var obj = {
    a: 2,
    foo: foo,
}
obj.foo();  // 2

这里 this 指向了 obj 对象.

对象属性引用链中只有最后一层会影响调用位置, 例如:

function foo() {
    console.log( this.a )
}

var obj1 = {
    a: 2,
    foo: foo,
}

var obj2 = {
    a: 15,
    obj1: obj1,
}

obj2.obj1.foo();  // 2

隐式绑定丢失

隐式绑定的函数丢失绑定对象时会使用默认绑定将 this 绑定到全局对象中. 例: 

function foo() {
    console.log( this.a )
}

var obj1 = {
    a: 2,
    foo: foo,
}

var bar = obj.foo
var a = "hello world"
bar() // "hello world"

此时 bar 是 obj.foo 的一个引用, 在全局环境中调用 bar 时没有任何修饰, 所以使用了默认绑定. 同样的情形发生在当把函数作为参数传递时(回调函数)时.

显示绑定

使用 call, apply 或者 bind 方法可以自行指定函数调用时的上下文. 

function foo() {
    console.log( this.a )
}

var obj1 = {
    a: 2,
}

foo.call(obj1) // 2

tips:
call 和 apply 的区别, 调用时传递参数的方式不同, apply 调用时传递的是参数数组

new 绑定

new 是调用对象构造函数时使用的关键字, 使用 new 来调用函数时, 会创建一个全新的对象.这个新对象会绑定到调用的构造函数中的 this 上. 

 function foo(a) {
    this.a = a
}

var bar = new foo(2)
console.log(bar.a) // 2

四种规则的优先级

new > 显示绑定 > 隐式绑定 > 默认绑定
